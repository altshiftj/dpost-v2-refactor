
# Device Watchdog Application

A Python application designed to monitor directories for new device-generated files, validate and rename them according to a specified naming convention, extract metadata from TIFF files, and synchronize files with a database using the `kadi_apy` API. The application also manages user sessions and archives processed files.

## Overview

The **Device Watchdog Application** monitors a specified directory for new files generated by devices (e.g., SEM). It validates the filenames against a required naming convention and, if necessary, prompts the user to rename them through a GUI dialog. The application extracts metadata from TIFF files, saves it as JSON, and synchronizes the data with a database. It also manages user sessions and archives processed files after synchronization.

---

## Features

- **Directory Monitoring**: Watches a directory for new files using `watchdog`.
- **Filename Validation**: Checks filenames against a specified regex pattern.
- **GUI Prompts**: Uses `tkinter` to prompt users for information when filenames are invalid.
- **Metadata Extraction**: Extracts and flattens metadata from TIFF files, including embedded XML data.
- **Session Management**: Automatically starts a session when files are processed and ends it after a timeout or when the user clicks "Done."
- **Database Synchronization**: Uploads files and metadata to a database using `kadi_apy`.
- **Archiving**: Moves processed files to an archive directory after synchronization.
- **Logging**: Logs important events and errors to a log file.

## Usage

1. **Prepare the Directories**:
   Ensure the `watch_dir` and its subdirectories (`To_Rename`, `Validated`, `Archive`) exist. The script will create them if they do not.

2. **Run the Application**:
   ```bash
   python your_script.py
   ```

3. **Monitoring Files**:
   - Place files generated by your device into the `watch_dir`.
   - The application will automatically detect new files.

4. **Filename Validation**:
   - If a file does not match the required naming convention, a GUI dialog will prompt you to provide the necessary information.
   - If you cancel the dialog, the file will be moved to the `To_Rename` folder.

5. **Session Management**:
   - A session starts when files are processed.
   - A dialog with a "Done" button appears, allowing you to manually end the session.
   - The session automatically ends after the specified timeout if not ended manually.

6. **Synchronization and Archiving**:
   - Upon session end, the application synchronizes files with the database using `kadi_apy`.
   - Processed files are then moved to the `Archive` directory.

## File Naming Convention

The required filename pattern is:

```plaintext
DeviceName_Institute_Name_DataQualifier_YYYY-MM-DD
```

**Example**: `SEM_IPAT_MuS_Cathode-20XD6-SO4_2023-10-19.tif`

**Components**:
- `DeviceName`: Name of the device (e.g., `SEM`)
- `Institute`: Short code for the institute (e.g., `IPAT`)
- `Name`: User's name or identifier (e.g., `MuS`)
- `DataQualifier`: Sample name or data qualifier (e.g., `Cathode-20XD6-SO4`)
- `YYYY-MM-DD`: Date in year-month-day format

## Metadata Extraction

- **TIFF Files**: Extracts metadata using `tifffile`.
- **Embedded XML**: Parses and flattens embedded XML metadata (e.g., `FEI_TITAN` tag).
- **Hashing**: Generates an MD5 hash of the file for integrity checks.
- **JSON Output**: Saves extracted metadata as JSON files alongside the TIFF files in the `Validated` directory.

---

## Session Management

- **Automatic Sessions**: Starts when the first file is processed.
- **Session Timeout**: Ends automatically if no new files are processed within the timeout period.
- **Manual Termination**: A "Done" button allows users to end the session manually.

**Actions on Session End**:
1. Synchronizes files with the database.
2. Archives processed files.

---

## Logging

- **Log File**: All events and errors are logged to `watchdog.log`.
- **Logging Levels**:
  - `INFO`: General events (e.g., file detection, renaming actions).
  - `WARNING`: Potential issues (e.g., invalid data during metadata extraction).
  - `ERROR`: Errors that prevent an action from completing.
