# Security Policy

## Supported Versions

Security fixes are currently applied to the latest `main` branch and the latest
published release line.

| Version | Supported |
| --- | --- |
| Latest release line | Yes |
| Older release lines | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities in public GitHub issues.

Use a private reporting channel:

1. Open a GitHub Security Advisory for this repository (preferred), or
2. Contact project maintainers through a private maintainer channel.

When reporting, include:

- A clear description of the issue and impact
- Reproduction steps or proof of concept
- Affected versions/commit range (if known)
- Any proposed mitigation

## Response Expectations

The maintainers will:

- Acknowledge receipt as quickly as practical
- Investigate and triage severity
- Coordinate a fix and disclosure timeline
- Credit reporters unless they request otherwise

We aim for coordinated disclosure and will share remediation guidance once a
fix is available.
